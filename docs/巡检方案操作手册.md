# AI 智能巡检系统 - Demo 演示流程

## 🎯 演示目标

展示 AI 智能巡检系统如何通过计算机视觉技术自动识别巡检点位的人员违规行为，实现从发现、分析到记录的完整闭环。

---

## 📋 演示准备

### 前置条件
- ✅ 系统已启动（`npm run dev:full`）
- ✅ 已配置飞书应用权限和 API Key
- ✅ 已连接飞书多维表格数据源
- ✅ 网络连接正常

### 演示环境
- 访问地址：http://localhost:3000
- 业务维度：人员违规数据表

---

## 🎬 演示流程

### 第一步：进入系统

1. 打开浏览器，访问系统地址
2. 进入 AI 智能巡检系统主界面
3. 确认顶部导航栏显示"人员违规数据表"

---

### 第二步：选择巡检点位

**操作说明：**
- 在左侧 AI 分析工作台中，找到点位选择下拉框
- 从下拉菜单中选择一个巡检点位

**可选点位：**
- 东门卫-仓库
- 生产车间A区
- 员工休息区
- 物资储备库
- 设备维修室

**演示建议：**
- 首次演示建议选择"生产车间A区"或"员工休息区"
- 这些点位有丰富的图像数据，更容易展示违规识别效果

---

### 第三步：启动 AI 分析

**操作说明：**
- 点击"开始分析"按钮
- 系统开始执行 AI 巡检分析流程

**预期效果：**
- 按钮显示加载状态
- 消息区域开始显示 AI 思考过程

---

### 第四步：观察 AI 思考过程

**阶段 1：调用工作流**
```
🤔 正在思考并生成答案...

📋 调用人员违规判断工作流
```
*延迟约 1-2 秒*

**阶段 2：查询图像数据**
```
🤔 正在思考并生成答案...

📋 调用人员违规判断工作流
📍 正在查询点位图像数据
```
*延迟约 1-2 秒*

**阶段 3：AI 视觉分析**
```
🤔 正在思考并生成答案...

📋 调用人员违规判断工作流
📍 正在查询点位图像数据
🔍 正在进行 AI 视觉分析
```
*延迟约 2-3 秒*

**阶段 4：生成分析报告**
```
🤔 正在思考并生成答案...

📋 调用人员违规判断工作流
📍 正在查询点位图像数据
🔍 正在进行 AI 视觉分析
📊 正在生成分析报告
```
*延迟约 1-2 秒*

**演示要点：**
- 每个步骤之间有自然的延迟，模拟真实思考过程
- 逐步展示让用户清晰了解 AI 分析流程
- 可以暂停讲解每个步骤的含义

---

### 第五步：查看违规识别卡片

**卡片展示内容：**

**📌 卡片头部**
- 标题：🚨 AI 巡检违规识别
- 背景：橙红色渐变

**🖼️ 图像区域**
- 显示违规现场图片
- 右上角标签显示违规类型（如：在岗玩手机）

**🚨 违规信息卡片（红色）**
- 类型：在岗玩手机 / 不符合5s标准 / 睡岗 / 走路玩手机 / 无违规
- 置信度：85%-97%
- 描述：详细的违规描述

**📍 位置信息卡片（蓝色）**
- 点位：选择的巡检点位名称
- 部门：生产部 / 行政部 / 设备部

**⏰ 时间信息卡片（紫色）**
- 日期：分析日期（格式：YYYY/MM/DD）
- 抓取时间：具体时间（格式：YYYY-MM-DD HH:mm:ss）

**🔘 操作按钮**
- "查看详细记录"按钮（橙红色渐变）

**演示要点：**
- 卡片设计精美，信息层次清晰
- 颜色编码帮助快速识别信息类型
- 置信度显示让用户了解识别准确度

---

### 第六步：自动记录写入

**如果检测到违规：**
- 系统自动将记录写入「人员违规数据表」
- 右侧表格自动刷新，显示新增的违规记录
- 显示确认消息：✅ 违规记录已自动写入「人员违规数据表」

**演示要点：**
- 展示数据自动流转到多维表格
- 强调闭环管理的优势
- 可以在表格中查看新增的记录详情

---

### 第七步：查看多维表格

**表格视图：**
- 查看所有巡检记录
- 展示表格的实时更新
- 可以点击单元格编辑内容

**应用视图：**
- 切换到应用视图模式
- 以卡片形式展示记录
- 更直观地查看图像和关键信息

**演示要点：**
- 展示数据的多维度管理
- 说明表格和视图的切换功能

---

### 第八步：多点位演示（可选）

**重复分析其他点位：**
1. 选择不同的巡检点位（如"员工休息区"）
2. 再次点击"开始分析"
3. 观察不同点位的分析结果

**演示要点：**
- 展示系统支持多点位巡检
- 不同点位可能有不同的违规类型
- 说明系统的通用性和扩展性

## ⚠️ 演示注意事项

### 成功演示的关键点

1. **网络稳定**：确保网络连接正常，AI 服务调用顺畅
2. **数据准备**：选择的点位需要有对应的图像数据
3. **时间控制**：整个演示流程约 2-3 分钟
4. **节奏把握**：在每个步骤适当停顿，给观众理解时间

### 可能遇到的问题

**问题 1：分析失败**
- 检查网络连接
- 确认 API 配置正确
- 切换到其他点位重试

**问题 2：识别不准确**
- 说明 AI 识别结果仅供参考
- 可以人工编辑记录
- 强调系统持续优化中

**问题 3：响应速度慢**
- 说明 AI 分析需要一定时间
- 正常情况下 5-10 秒完成
- 可以提前准备多个点位数据
