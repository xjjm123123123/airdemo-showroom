# AirDemo - Feishu AI Solution Showroom（One‑page）

## 产品定位
AirDemo 是一个“飞书 AI 解决方案样板间”前端演示应用，用于在短时间内通过结构化数据表与 AI 分析联动，向客户展示：从业务数据采集 → AI 洞察 → 行动闭环的完整链路。

## 核心功能
### 1) 解决方案目录（Catalog）
- 以卡片形式展示 Demo 列表，包含价值主张、适用人群与要点。
- 点击卡片进入对应 Demo 的演示工作区。

### 2) 左侧工具栏（App Switcher）
- 用于切换不同“工具形态”的解决方案演示：
  - `aPaaS 低代码`：展示低代码/自动化类能力入口。
  - `多维表格 Base`：进入结构化数据表与多视图联动的核心演示区。
  - `提示词模板`：进入提示词模板库，一键调用预设分析指令。

### 3) 多维表格工作区（Base Workspace）
- 内置表格 UI：支持横向滚动、表头吸顶、首列/末列粘性定位。
- 表格下方提供可拖动的水平滚动条，便于在列数较多时快速移动视窗。
- 数据分为主表与巡检子表：
  - 主表（示例：人员违规明细）
  - 巡检子表视图（示例：巡检管理/设备巡检/厂区巡检）可一键切换

### 4) 表格编辑能力
- 表格编辑用于快速贴合客户真实数据结构，便于现场按需求做定制化方案演示。
- 新增记录：一键添加新行。
- 单元格编辑：点击进入编辑态，失焦或回车提交。
- 新增字段：可配置字段类型（文本/数字/选项/图片）并动态加入列。
- 删除记录/删除字段：支持删除行与字段（保留 id 列）。

### 5) 巡检多视图切换
- 支持底部“Views”面板切换不同业务视图：
  - 人员违规数据表
  - 巡检管理数据表
  - 设备巡检数据表
  - 厂区巡检数据表
- 每个视图可绑定独立的子表数据源，切换时表结构与内容同步更新。

### 6) Aily 分析工作台（右侧对话）
- 右侧提供对话区，模拟“业务分析师 Aily”。
- 发送问题后，会将“当前表格结构 + 当前可见数据”作为上下文，生成结果导向的中文分析输出。
- 左侧 `提示词模板` 提供内置模板库：
  - 深度合规风险诊断
  - 管理层决策简报
  - 巡检布控优化建议

## 数据展示细节
- 图片字段：以缩略图展示，适合展示巡检抓拍/监控截图。
- 状态/风险等枚举字段：以胶囊标签展示，强化“可视化状态”。
- 完成率：支持按百分比字符串展示（如 80%、100%）。

## 典型演示路径（5 分钟脚本建议）
1. 进入 Catalog，快速对齐业务价值与客户画像。
2. 进入巡检 Demo：先看人员违规主表，强调“数据结构化”。
3. 切换巡检管理/设备巡检/厂区巡检子表，强调“多维视角对齐管理动作”。
4. 通过 Aily 提问：让 AI 输出风险点、整改建议、优先级与下一步行动。
5. 演示表格编辑：新增字段/修改状态，强调“数据闭环可落地”。

## 本地运行
- 安装依赖：`npm install`
- 启动开发：`npm run dev`
